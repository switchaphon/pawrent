## Lessons Learned

### What Worked

1. **Validation-first pipeline caught 3 gaps** ‚Äî `getActiveSOSAlertForPet()` was missing from the rename list, `e2e/auth-flow.spec.ts` was omitted, and UI text strings had no mapping. All fixed before execution.
2. **Atomic commit strategy was correct** ‚Äî single commit makes rollback trivial (`git revert`).
3. **Blast radius estimate was accurate** ‚Äî PRP predicted ~35 files, actual was 36.
4. **`.next` cache cleanup** ‚Äî stale `.next/types/` references caused phantom type errors after route renames. `rm -rf .next` before type-check is necessary for route moves.

### What Didn't Work

1. **PRP missed `getActiveSOSAlertForPet()`** ‚Äî this function existed in `lib/db.ts` but wasn't listed in the rename table. Validation caught it.
2. **PRP missed `e2e/auth-flow.spec.ts`** ‚Äî had `/sos` in a protected routes array. Easy to miss because it's a different file from `authenticated-flows.spec.ts`.
3. **UI text strings weren't specified** ‚Äî the PRP said "rename" but didn't specify what SOS UI text should become. Added a UI Text Mapping table during refinement.
4. **Geospatial param type renames omitted** ‚Äî `NearbyAlertsParams`, `AlertsWithinBboxParams` types in geospatial.ts weren't in the naming table.

### Add to Future PRPs

1. **Always grep for the old name across the entire codebase** before writing the rename table ‚Äî don't rely on memory of which files use it.
2. **Include UI text mapping** for any rename that touches user-facing strings.
3. **Note `.next` cache invalidation** as a required step for route renames.
4. **CommitLint lowercase rule** ‚Äî commit subjects must be fully lowercase; acronyms like "SOS" trigger `subject-case` failure.
