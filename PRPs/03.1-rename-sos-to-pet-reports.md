## Problem

The codebase uses `sos_alerts` naming throughout â€” tables, types, functions, components, routes, tests. With the pivot to a unified community hub at `/post/*` covering lost/found/returned/social, the "SOS" naming is misleading and inconsistent. Renaming now (before PRP-04 builds new features on top) avoids building new code on legacy naming and prevents a much larger migration later.
