# PRP-04.1: Poster & Social Share Card Generation

## Priority: HIGH

## Prerequisites: PRP-04 (lost alert detail page, alert data model)

## Blocks: None (standalone enhancement)

## Problem

Physical posters on telephone poles and temple bulletin boards are still the most effective pet recovery tool in Thai neighborhoods. Digital-only alerts miss the aunties, security guards, and motorcycle taxi riders who don't use LINE OA. A poster with a QR code bridges offline and online — strangers scan the QR, land on the live alert page, and can report a sighting instantly.

Additionally, pet owners need a social-optimized share card (JPEG) to post on Facebook groups, Instagram stories, TikTok, and X — the primary channels Thai pet parents use for lost pet broadcasts.

---

## Scope

**In scope:**

- A4 PDF poster generation (print-quality, bilingual Thai + English)
- Social share card JPEG generation (1080x1350 for IG/FB, 1080x1080 square variant)
- QR code linking to live alert page (`/post/[id]`)
- Server-side generation via API route
- "Generate Poster" button on alert detail page (`app/post/[id]/page.tsx`)
- "Download Share Card" button for social posting
- Both formats downloadable

**Out of scope:**

- Print shop integration / routing
- Partner print network
- Automatic social media posting (user downloads and posts manually)

### 4.1.2 A4 PDF Poster Layout (Thai-style — bold, emotional, high-contrast)

Visual reference: Thai lost pet posters use HUGE text, bold colors (yellow/red), and emotional language. Not corporate/clean — loud and urgent. See competitive screenshots for reference.

- [ ] **"แมวหาย!" / "หมาหาย!"** header in MASSIVE bold red text (species-aware: cat/dog)
- [ ] **Pet name** in huge text right below header
- [ ] **Reward amount** in huge red/gold text (if set): "💰 รางวัล ฿XX,XXX" — second most prominent element
- [ ] **Large pet photo** (best quality from `photo_urls` — full width hero, high res)
- [ ] **Lost date & location**: "📅 หายวันที่ 13 เมษายน 2569" + "📍 หมู่บ้านอริสรา 2 บางบัวทอง" (from `location_description`)
- [ ] **Pet details**: sex ("เพศผู้/เมีย"), neutered ("ทำหมันแล้ว"), breed, color
- [ ] **Distinguishing marks** section with bullet points (parsed from free-text, line-break separated)
- [ ] **Contact section**:
  - If `contact_phone` provided (opt-in): show phone in HUGE text at bottom (like real Thai posters — this is the #1 action item for finders)
  - Always: QR code linking to `/post/[id]` + "สแกนเพื่อดูรายละเอียดเพิ่มเติม" (scan for more details)
  - Always: LINE OA QR or ID for platform contact
- [ ] **Emotional CTA**: "หากพบเห็นน้อง ขอความเมตตาโทรแจ้งเจ้าของด้วยนะคะ 🙏" (If you see this pet, please kindly call the owner)
- [ ] **Background**: Yellow or white with red/orange accents (eye-catching, matches Thai poster norms)
- [ ] Pawrent branding / logo (small, bottom corner — not dominant)
- [ ] Bilingual: Thai primary, English secondary (small text)

### 4.1.3 Social Share Card (JPEG) — Thai visual style

Same bold visual language as poster, optimized for FB groups / IG stories / LINE groups.

- [ ] Create `app/api/share-card/[alertId]/route.ts` — generates JPEG
- [ ] Dimensions: 1080x1350 (IG portrait) and 1080x1080 (square) variants
- [ ] Optimized for social: bold text, high contrast, readable on mobile thumb-scroll
- [ ] Layout:
  - **"แมวหาย!" / "หมาหาย!"** banner at top (red background, white text, BOLD)
  - **Pet photo** (prominent, ~50% of card)
  - **Pet name** in large text
  - **Reward amount** in red/gold (if set)
  - **Lost date + location** (from `location_description`)
  - **Key details**: breed, color, sex, neutered (single line)
  - **Distinguishing marks** (truncated to 2-3 lines)
  - **Contact**: phone number if opt-in, otherwise "สแกน QR เพื่อติดต่อ"
  - **QR code** (bottom corner, links to `/post/[id]`)
  - **"ช่วยแชร์ด้วยนะ 🙏"** CTA at bottom
  - Pawrent branding (small)
- [ ] Return as JPEG blob (quality 90%)
- [ ] **Note:** Share card should look like the viral Thai lost pet posts on Facebook — not a corporate infographic

### 4.1.4 Library Selection (platform-agnostic — no vendor lock-in)

- [ ] **PDF poster:** `pdf-lib` (lightweight, programmatic, no React dependency, works in any Node.js runtime)
- [ ] **JPEG share card:** `sharp` (image compositing, text overlay, fast, widely supported) + `@napi-rs/canvas` (for complex text layout with Thai font rendering if `sharp` text is insufficient)
- [ ] **QR code:** `qrcode` (generates QR as PNG buffer, composited into poster/card via `sharp`)
- [ ] **Thai font:** Bundle `Sarabun` or `IBM Plex Sans Thai` `.ttf` for `pdf-lib` embedding and `sharp`/`canvas` text rendering
- [ ] Install chosen dependencies

> **Why not `@vercel/og`:** Ties to Vercel edge runtime. `sharp` + `pdf-lib` are platform-agnostic — works on Vercel, AWS, self-hosted, Docker.
> **Why not `@react-pdf/renderer`:** Heavy dependency tree, slower cold start. `pdf-lib` is ~10x lighter for programmatic layout.
