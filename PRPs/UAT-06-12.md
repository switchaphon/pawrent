## Test 1: Lost Pet Push Notification

> **Goal:** When Tester A reports a lost pet, Tester B (nearby) gets a LINE message.

### Prepare Test Data (one-time)

1. Open **Supabase Dashboard → Table Editor → `profiles`**
2. Find **Tester B's row** (search by `line_display_name`)
3. Edit these columns:

| Column | Value to set | Why |
|--------|-------------|-----|
| `notification_radius_km` | `5` | Opt-in for push (5km radius) |
| `push_species_filter` | `{dog,cat}` | Receive alerts for both dogs and cats |
| `push_quiet_start` | *(leave empty)* | No quiet hours for this test |
| `push_quiet_end` | *(leave empty)* | |
| `home_geog` | `SRID=4326;POINT(100.5018 13.7563)` | Central Bangkok (Siam area) |

4. Click **Save**

### Prepare Supabase Webhook (one-time)

1. Open **Supabase Dashboard → Database → Webhooks**
2. Click **Create new webhook**
3. Fill in:
   - Name: `push-on-new-alert`
   - Table: `pet_reports`
   - Events: `INSERT`
   - URL: `https://{staging_url}/api/alerts/push`
   - Headers: `Authorization: Bearer my-push-secret-2026` (must match your `PUSH_WEBHOOK_SECRET`)
4. Click **Create**

### Run the Test

1. **On Phone A** — open the staging LIFF app via LINE
2. Go to **"แจ้งหาย"** (Report Lost) page
3. Fill in the lost pet form:
   - Pet: pick any registered pet (or create one named "มะลิ", breed "พุดเดิ้ล")
   - Location: **tap the map near Siam Paragon** (lat ~13.7463, lng ~100.5327 — within 5km of Tester B)
   - Reward: `500`
   - Description: "หายแถวสยาม"
4. Submit the form
5. **Wait 30 seconds**
6. **On Phone B** — check LINE chat with Pawrent OA

### What to Check

- [ ] **Phone B receives a LINE Flex Message** within 30 seconds
- [ ] Message has a **red header** with text **"🚨 สัตว์เลี้ยงหาย"**
- [ ] **Pet photo** is visible (4:3 ratio, not stretched)
- [ ] Shows **breed and sex** (e.g., "พุดเดิ้ล · เพศผู้")
- [ ] Shows **distance** (e.g., "📍 1.2km จากคุณ")
- [ ] Shows **lost date** in Thai: "📅 หายวันที่ 14 เม.ย. 2569"
- [ ] Shows **reward**: "💰 รางวัล ฿500"
- [ ] **Tap "ฉันเห็นน้อง!" button** → opens the alert detail page in LIFF
- [ ] Open **Supabase → Table Editor → `push_logs`** → new row exists with `recipient_count >= 1`

## Test 16: Database Integrity

> **Goal:** Users can't read each other's health data.

1. Open **Supabase → SQL Editor**
2. Run this query (uses the **anon key**, not service role):

```sql
-- Switch to anon role to simulate a different user
SET ROLE anon;
SELECT * FROM pet_milestones LIMIT 5;
SELECT * FROM health_reminders LIMIT 5;
SELECT * FROM pet_weight_logs LIMIT 5;
SELECT * FROM push_logs LIMIT 5;
```

- [ ] `pet_milestones` — returns 0 rows (RLS blocks)
- [ ] `health_reminders` — returns 0 rows
- [ ] `pet_weight_logs` — returns 0 rows
- [ ] `push_logs` — returns 0 rows

- [ ] All related rows are gone (CASCADE worked)
